# Bootstrap Script Usage (SOP v3.0)

## Quick Start

```bash
# 1. Copy script to where you want to create projects
cp bootstrap.sh ~/Project/

# 2. Make executable
chmod +x ~/Project/bootstrap.sh

# 3. Run
cd ~/Project
./bootstrap.sh
```

## What it creates

```
[project-name]/
├── README.md              <- AI entry point
├── SOP.md                 <- Full SOP v3.0
├── PROJECT.md             <- Project knowledge base
├── SERVER_RULES.md        <- Hosting rules
├── index.php              <- Router (maintenance/live)
├── .htaccess              <- Routing rules
├── .gitignore             <- SOP-compliant
├── docker-compose.yml     <- Docker config
├── wp-config-local.php    <- Local DB config
├── wp-config-production.php <- Template for hosting
├── wp/
│   └── wp-content/
│       ├── themes/
│       ├── mu-plugins/
│       └── plugins/
├── maintenance/
│   └── index.html         <- Coming Soon page
├── backups/               <- DB dumps folder
└── docs/                  <- Additional docs
```

## AI Reading Order

```
README.md → SOP.md → PROJECT.md
```

README.md is the entry point that tells AI where to go next.

## Notes

- Bootstrap is for OWNERS, not AI
- AI never runs bootstrap
- After bootstrap, owner initializes Git and pushes
- Then AI reads README.md first when starting work
